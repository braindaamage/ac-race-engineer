# Implementation Plan: Garage Views Data Population

**Branch**: `036-garage-views-data` | **Date**: 2026-03-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/036-garage-views-data/spec.md`

## Summary

Populate the Garage Home and Car Tracks placeholder views with real session data and Assetto Corsa asset metadata. Backend: add `track_config` column to sessions table, aggregation queries, AC metadata reader, image serving endpoints, and grouped data endpoints. Frontend: replace placeholder views with data-driven car/track card grids, add search filtering, contextual headers, and display name resolution in breadcrumbs.

## Technical Context

**Language/Version**: Python 3.11+ (backend), TypeScript strict (frontend)
**Primary Dependencies**: FastAPI, Pydantic v2 (backend); React 18, TanStack Query v5, react-router-dom v7 (frontend)
**Storage**: SQLite via stdlib sqlite3 (single file: data/ac_engineer.db)
**Testing**: pytest (backend, ~1055 tests), Vitest + Testing Library (frontend, ~402 tests)
**Target Platform**: Windows desktop (Tauri v2 shell)
**Project Type**: Desktop app (Tauri + React frontend, FastAPI backend sidecar)
**Performance Goals**: Garage Home loads in <2s, image serving with browser caching
**Constraints**: AC install path may not be configured — all metadata reads must have graceful fallbacks
**Scale/Scope**: Typically <50 cars, <50 tracks per car, <100 sessions per car+track
