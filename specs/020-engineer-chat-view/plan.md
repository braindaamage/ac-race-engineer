# Implementation Plan: Engineer Chat View

**Branch**: `020-engineer-chat-view` | **Date**: 2026-03-06 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/020-engineer-chat-view/spec.md`

## Summary

Build the Engineer Chat view for the AC Race Engineer desktop application — a conversational interface where the driver interacts with the AI race engineer. The view supports full session analysis (producing structured recommendation cards), free-form follow-up questions, and one-click application of setup changes. Built with React + TypeScript + TanStack Query + Zustand, using existing Phase 6 backend endpoints and Phase 7.3 WebSocket infrastructure.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode) with React 18
**Primary Dependencies**: React 18, TanStack Query v5, Zustand v5, Vite, Vitest
**Storage**: N/A (frontend only — all persistence via backend API)
**Testing**: Vitest + React Testing Library
**Target Platform**: Windows desktop (Tauri v2 shell)
**Project Type**: Desktop app frontend view
**Performance Goals**: Responsive chat UX — messages render instantly, auto-scroll smooth
**Constraints**: No localStorage/sessionStorage, no hardcoded colors, no explicit `any`, JetBrains Mono for numeric data, all colors via design tokens
**Scale/Scope**: 1 view (~11 new component files), 2 new hooks, ~7 test files, type additions to 1 existing file
