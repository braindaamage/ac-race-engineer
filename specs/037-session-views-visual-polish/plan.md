# Implementation Plan: Session Views Visual Polish

**Branch**: `037-session-views-visual-polish` | **Date**: 2026-03-15 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/037-session-views-visual-polish/spec.md`

## Summary

Visual redesign of session detail views (Lap Analysis, Setup Compare, Engineer), Settings view, and a new session detail header component. All changes are frontend-only: CSS updates, minor JSX structure adjustments for styling, and one new component (SessionHeader). No backend changes, no new API endpoints, no changes to hooks or business logic. The goal is visual consistency with the design language established in Phases 14.1 (navigation shell, design tokens) and 14.2 (Garage and Tracks views).

## Technical Context

**Language/Version**: TypeScript strict (frontend only)
**Primary Dependencies**: React 18, react-router-dom v7, TanStack Query v5, Zustand v5, Recharts v3, Vitest + Testing Library
**Storage**: N/A (no backend changes)
**Testing**: Vitest + Testing Library (~431 existing frontend tests + new SessionHeader tests)
**Target Platform**: Windows desktop (Tauri v2 shell)
**Project Type**: Desktop app (Tauri + React frontend)
**Performance Goals**: No performance regressions from CSS changes
**Constraints**: All colors from design tokens (Principle XII), JetBrains Mono for numeric data, no business logic changes, both themes must work
**Scale/Scope**: ~15 CSS files modified, 1 new component, 1 new test file

## Complexity Tracking

No constitution violations to justify. All changes align with existing patterns.

| Decision | Why | Simpler Alternative Rejected Because |
|----------|-----|-------------------------------------|
| Hardcoded hex in Recharts stroke props | Recharts requires string hex values, not CSS variables | Using getComputedStyle() adds runtime complexity and potential flicker |
| color-mix() for user message timestamp | Replaces hardcoded rgba() while staying theme-aware | Adding a new --text-on-brand token adds complexity for one usage |
